ZWARMIS BACKEND - QUICK REFERENCE CARD
=====================================

🚀 START SERVER
===============
Windows:  double-click start.bat  OR  npm start
Linux:    ./start.sh              OR  npm start
Server runs on: http://localhost:3001

📋 KEY ENDPOINTS
================
GET    /health                           - Server health
GET    /api/admin/dams                   - Get all dams
GET    /api/admin/dams/:damId            - Get one dam
GET    /api/auth/status                  - Auth status
POST   /api/auth/request-access          - Request access
POST   /api/auth/verify-code             - Verify code (login)
POST   /api/admin/update-dam             - Update dam (auth required)
GET    /api/admin/history                - Get all history
GET    /api/admin/history/:damId         - Get dam history
POST   /api/auth/logout                  - Logout

🔐 AUTHENTICATION FLOW
======================
1. POST /api/auth/request-access
   Body: {username, userEmail, company, purpose}
   → Check console for verification code

2. POST /api/auth/verify-code
   Body: {code}
   → Get sessionId from cookie

3. Use sessionId in cookies for authenticated requests
   → Session expires after 30 minutes

4. POST /api/auth/logout
   → Destroy session

📊 DAM UPDATE
=============

🛠️ TROUBLESHOOT
================
Port in use:
  Windows: netstat -ano | findstr :3001
           taskkill /PID <PID> /F
  Linux:   lsof -i :3001
           kill -9 <PID>

CORS error: Check CORS_ORIGIN in config.js
No data: Check dams-data.json exists
Email: Demo mode prints codes to console

🔍 DEMO MODE
============
Default behavior:
- Verification codes printed to console
- No real emails sent
- Data in memory only
- Perfect for testing

To enable real email:
1. Get Gmail App Password
2. Set SMTP_USER and SMTP_PASS in .env
3. Restart server

💾 DATA STORAGE
===============
Current: In-memory (lost on restart)
Production: MongoDB, PostgreSQL, etc.

Sessions: HTTP-only cookies (30 min)
Codes: 6 digits, expire in 10 min, max 5 attempts
Users: Stored after verification
